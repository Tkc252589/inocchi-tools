4 技術構成 / ファイルの役割と連携

Chrome拡張の各ファイルがどんな役割を持ち、どう連携しているかの図解です。

📦 manifest.json
拡張の設定ファイル（Manifest V3）
  • permissions: storage, tabs, scripting
  • host_permissions: carousell.sg, mercari.com, mercdn.net, mercari-shops-static.com
  • content_scripts: /sell* → carousell.js, /p/* → listed.js
⚙️ background.js
Service Worker（バックグラウンド処理）
  • smartFetchImages() — URLに応じて画像取得方式を選択
  • fetchImagesFromUrl() — 連番URL方式（通常メルカリ）
  • fetchImagesViaTab() — タブ方式（Shops）
  • updateSheet() — GAS APIでスプシ更新
  • downloadImages() — 画像ダウンロード共通処理
🎨 content-carousell.js
Carousell出品ページ用 Content Script
  • selectDropdown() — ドロップダウン操作（Category/Brand/Type共通）
  • fillForm() — 9項目の自動入力
  • uploadOnly() — 画像アップロードのみ
  • detectType() — タイトルからバッグ種類を推定
  • Brand未登録時: Others → 新input検出 → ブランド名入力
✅ content-listed.js
出品完了ページ用 Content Script
  • URLから商品IDを自動抽出（/p/slug-12345/）
  • background.jsに更新リクエスト送信
  • 5秒後にステータス自動非表示
🖥️ popup.html / popup.js
ポップアップUI（ピンクデザイン）
  • Google Visualization API で未出品商品を取得
  • SQLクエリ: A列・B列が空 & N列にタイトルあり
  • ブランド・Type自動検出して一覧表示
  • 「出品→」で chrome.storage にデータ保存 → タブ作成
🔧 CarousellGAS.gs
スプシ書き戻し用 GAS Web App
  • POST { action: "markListed", title, carousellId, url, date }
  • N列のタイトルで行を検索
  • A列=日付, B列=ID, C列=URL を書き込み
ファイル間の連携フロー
popup.js→background.jsOPEN_CAROUSELL メッセージ + 商品データ
background.js→chrome.storagependingListing / lastListedItem を保存
chrome.storage→content-carousell.js保存されたデータを読み取り → UI表示
content-carousell.js→background.jsFETCH_IMAGES メッセージ（画像取得依頼）
background.js→メルカリ画像URL取得 & ダウンロード
content-listed.js→background.jsUPDATE_SHEET メッセージ（ID + URL）
background.js→GAS Web AppPOST リクエスト → スプシ書き込み
