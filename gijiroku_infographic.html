AI議事録自動化ツール

会議を録音 → AI が文字起こし → 議事録 + アクションアイテムを自動生成

こんな悩みありませんか？
  • 会議中にメモを取ると議論に集中できない
  • 議事録を書く時間がもったいない（1時間の会議 → 30分の議事録作成）
  • 「あれ、何が決まったんだっけ？」とアクションアイテムが曖昧に
  • 議事録の共有を忘れて、決定事項が宙に浮く
Before / After
❌ Before（手動）
  • 会議中に必死にメモ
  • 終了後30分かけて議事録作成
  • アクションアイテムを思い出す
  • Google Docsにコピペ
  • 関係者に共有メール
✅ After（自動化）
  • 「録音開始」をダブルクリック
  • 会議に100%集中
  • 「録音停止」をダブルクリック
  • 数分待つだけで全部完成！
  • 議事録 + アクションアイテム + PDF

セットアップ手順（初回のみ）
1

Homebrew + 必要ソフトのインストール

brew install python@3.12 ffmpeg brew install --cask blackhole-2ch
2

Python仮想環境の作成

mkdir -p ~/.local/share/gijiroku /opt/homebrew/bin/python3.12 -m venv ~/.local/share/gijiroku/venv ~/.local/share/gijiroku/venv/bin/pip install mlx-whisper google-genai \ google-api-python-client google-auth-httplib2 google-auth-oauthlib
3

Audio MIDI設定でマルチ出力デバイスを作成

「Audio MIDI設定」→ 左下の＋ → マルチ出力デバイスを作成 → BlackHole 2ch + スピーカーにチェック → 名前を「会議録音」に変更

4

Gemini APIキー取得

https://aistudio.google.com/apikey → 「APIキーを作成」→ コピー

5

Google OAuth設定

Google Cloud Console → OAuthクライアントID作成（デスクトップアプリ）→ JSONダウンロード → Google Docs API / Drive API を有効化

6

Claude Code にツール作成を依頼

下の指示文をClaude Codeに貼り付けるだけ！

以下のファイル構成で議事録自動化ツールを作成してください。

場所: ~/Documents/議事録/

機能:
- BlackHole 2ch経由でシステム音声をWAV録音（16kHz、モノラル）
- mlx-whisper（mlx-community/whisper-large-v3-turbo）で日本語文字起こし
- Gemini 2.5 Flashで議事録整形・タイトル生成・参加者抽出・アクションアイテム抽出
- Google Docs APIで議事録をドキュメント化・PDF化・Google Driveに保存
- macOS通知センターで進捗通知
- 録音開始.command / 録音停止.command をダブルクリックで操作

必要なファイル:
- 録音開始.command / 録音停止.command（zshスクリプト）
- scripts/record_start.py, record_stop.py, transcribe.py
- scripts/formatter.py, google_docs.py, notify.py
- config/settings.json

venvの場所: ~/.local/share/gijiroku/venv/
config/client_secret.json に Google OAuthクライアントIDのJSONを手動配置
初回実行時にブラウザでGoogle認証が開きます。
ポイント
  • Mac（Apple Silicon M1〜M4）専用。Whisperがローカルで高速動作
  • APIコスト：Gemini 2.5 Flash使用で1回あたり約1円以下
  • 録音データはローカル処理→プライバシー安全
  • 長時間会議（2時間超）はメモリ消費大。1時間程度が最適
  • Google Docs + PDFが自動生成されるので共有も簡単
